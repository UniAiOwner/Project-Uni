# Project Uni Website Blueprint v1.0

## Core Direction

Project Uni is not presented as a normal startup, portfolio, or company website. The website is a cinematic digital experience where visitors feel they are entering the Project Uni universe and unlocking chapters of a larger mission.

The first build should prioritize architecture, story, emotion, trust, and journey. Implementation comes only after this blueprint is approved.

**Do not write code during this phase.** This document is the final website blueprint that must be approved before UI design, copywriting, or implementation starts.

## Experience Principle

The website should feel like an interactive journey, not a static scroll page.

Recommended chapter flow:

1. Welcome
2. The Problem
3. The Vision
4. The Future of Intelligence
5. UniSI
6. The Ecosystem
7. Research
8. Roadmap
9. Become Part of Project Uni

Each chapter should feel intentionally unlocked through motion, interaction, and story progression.

## First-Time Visitor Journey

A first-time visitor should move through three emotional states:

1. **Curiosity** — What is Project Uni, and why does it feel different?
2. **Understanding** — Project Uni is a Synthetic Intelligence ecosystem, not just another AI tool.
3. **Belonging** — This is a community-built future that the visitor can help shape.

The journey should never start with funding. Funding should appear only after the visitor understands the mission, trust model, roadmap, and human story.

## Story Strategy

### How We Explain the Vision

The vision should be explained as a future where intelligence is not locked inside a few companies. Project Uni should be described as a community-built Synthetic Intelligence ecosystem designed to help people, businesses, researchers, and creators.

### How We Introduce Synthetic Intelligence

Synthetic Intelligence should be introduced as a layered living digital system with memory, perception, cognition, emotion, execution, safety, and self-improvement. The site should avoid vague hype and instead explain what each layer does in simple terms.

### How We Introduce the Ecosystem

The ecosystem should be introduced after UniSI, because every product should feel like an expression of the same core intelligence. UniAiBiz, UniMed, UniTrade, UniGuard, UniGlow, Cognesis, and UniAIOS should not feel like random apps. They should feel like planets orbiting the UniSI core.

### How We Build Emotional Connection

The site should make the visitor feel that Project Uni is being built by real people with limited resources, high ambition, and a clear reason. The founder story, community invitation, transparent roadmap, and support program should carry this emotion.

### How We Create Trust

Trust should be built through:

- Clear claims and visible limitations.
- Transparent roadmap status.
- Safety notes for medical, finance, and security-related products.
- Founder identity and contact details.
- Public progress logs.
- Honest support language that avoids investment-like promises.

### Where Funding Appears

Funding should appear after Vision, UniSI, Ecosystem, Roadmap, and Transparency. Support is the result of belief, not the first ask.

### How Users Become Supporters

Users should be invited to become supporters through a dedicated Support Project Uni page and contextual CTAs after major story milestones. The language should frame support as participation in the mission, not ownership, investment, or guaranteed return.

## Global Motion and Interaction System

### Motion Personality

- Cinematic but fast.
- Futuristic but readable.
- Emotional but not childish.
- Premium but not cold.

### Core Effects

- Chapter unlock transitions.
- Neural grid / particle atmosphere.
- Orbiting ecosystem map.
- Signal pulses through UniSI architecture.
- Smooth page transitions.
- Scroll-progress chapter indicator.
- Hover states that reveal meaning, not just decoration.

### Accessibility Boundaries

- Respect reduced-motion settings.
- Keep contrast high.
- Avoid motion that blocks reading.
- Every interactive visual must have a text explanation.

## Design Inspiration Translation

The site should learn from major product experiences without copying them:

- Apple: cinematic simplicity and emotional reveal.
- OpenAI: clarity around intelligence and capability.
- Tesla: future-facing ambition.
- SpaceX: mission scale and launch energy.
- GitHub: community and builder credibility.
- Stripe: information architecture and trust.
- Linear: polish, speed, and product precision.
