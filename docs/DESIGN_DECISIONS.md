# Project Uni Website Design Decisions

## 1. Static v1, Designed Like a Product System

Version 1.0 remains a static HTML/CSS/JS website so the public experience is reviewable without a build pipeline. The structure is intentionally data-generated from `tools/build_static_site.py` to keep page architecture consistent while still allowing page-specific story content.

## 2. Hero Strategy

The homepage leads with “The Future of Intelligence Begins Here.” before the Project Uni brand name. This creates emotion first, then identity, matching the founder direction.

## 3. Support Is Present but Not Pushy

Support Project Uni remains visible in navigation and CTA bands, but content order prioritizes mission, trust, and understanding before asking for support.

## 4. Every Page Gets a Reason to Exist

Pages are not placeholders. Each page now includes a specific promise, sections, trust note, journey logic, and next action so visitors understand why the page exists.

## 5. Motion Supports Story

Motion is limited to reveal transitions, orbiting ecosystem cues, chapter progress, and hover states. The site respects reduced-motion preferences.

## 6. Trust Is a Product Feature

Transparency, safety notes, clear limitations, public roadmap, and support disclaimers are part of the main experience instead of hidden legal afterthoughts.
