# Project Uni Governance Mode

Project Uni implementation is owned as if by a founding engineering team, not as a prompt-by-prompt coding task.

## Operating Standard

Before creating or changing any page, ask:

1. Does this page help explain Project Uni?
2. Does this increase trust?
3. Does this improve user experience?
4. Does this strengthen the ecosystem?
5. Would Apple, OpenAI, or Stripe ship something with this level of intention?

If the answer is no, the work must be redesigned before shipping.

## Primary Mission

Create the best possible public experience for Project Uni: not the biggest, not the flashiest, but the most meaningful.

## Priorities

1. Project Uni Vision
2. User Experience
3. Trust
4. Storytelling
5. Funding Conversion

## Engineering Rules

- The blueprint is the law.
- The repository is the single source of truth.
- Every page must have a reason to exist.
- Every section must tell part of the story.
- Every animation must support the story.
- Every interaction must feel intentional.
- Treat every commit as production quality.
- Document important design decisions.
- Stop only when blocked by a technical issue.
